f:Tf36,<h2>El Problema de los Costos en BigQuery</h2>
<p>BigQuery es increíblemente poderoso, pero <strong>puede vaciar tu presupuesto</strong> si no sabés lo que estás haciendo.</p>
<p>En MercadoLibre, nuestras queries de analytics estaban generando <strong>$15,000 USD mensuales</strong> en costos. Después de aplicar estas técnicas, bajamos a <strong>$4,500 USD</strong>.</p>
<h2>Técnica #1: Particionamiento por Fecha</h2>
<p><strong>Antes:</strong></p>
<pre><code class="language-sql">SELECT * FROM transactions
WHERE DATE(created_at) >= '2026-01-01'
</code></pre>
<p><strong>Después:</strong></p>
<pre><code class="language-sql">SELECT * FROM transactions
WHERE _PARTITIONDATE >= '2026-01-01'
</code></pre>
<p><strong>Ahorro:</strong> 95% menos datos escaneados.</p>
<h2>Técnica #2: Clustering Inteligente</h2>
<pre><code class="language-sql">CREATE TABLE transactions
PARTITION BY DATE(created_at)
CLUSTER BY user_id, country_code
AS SELECT * FROM source_table
</code></pre>
<p><strong>Ahorro:</strong> 40% adicional en queries filtradas por user_id.</p>
<h2>Técnica #3: Materializar Queries Frecuentes</h2>
<p>Si ejecutás la misma query 100 veces al día, <strong>materializala</strong>.</p>
<pre><code class="language-sql">CREATE MATERIALIZED VIEW daily_aggregates AS
SELECT 
  DATE(created_at) as date,
  country_code,
  SUM(amount) as total_amount
FROM transactions
GROUP BY 1, 2
</code></pre>
<p><strong>Ahorro:</strong> 98% en queries repetidas.</p>
<h2>Técnica #4: Usa Tablas Externas para Staging</h2>
<p>Para datos que cambian poco, usa Google Cloud Storage:</p>
<pre><code class="language-sql">CREATE EXTERNAL TABLE staging_data
OPTIONS (
  format = 'PARQUET',
  uris = ['gs://bucket/data/*.parquet']
)
</code></pre>
<p><strong>Ahorro:</strong> $0 de storage en BigQuery.</p>
<h2>Técnica #5: Slots Reservados vs On-Demand</h2>
<p>Para workloads predecibles, <strong>slots reservados son más baratos</strong>.</p>
<ul>
<li>On-demand: $6.25 por TB escaneado</li>
<li>Flat-rate: $2,000/mes por 100 slots</li>
</ul>
<p>Si escaneás >320TB/mes, flat-rate gana.</p>
<h2>Técnica #6: Monitoreo de Costos</h2>
<pre><code class="language-python"># Script diario de monitoring
from google.cloud import bigquery

client = bigquery.Client()

query = """
SELECT 
  user_email,
  SUM(total_bytes_billed) / 1024 / 1024 / 1024 / 1024 as tb_billed,
  SUM(total_bytes_billed) * 6.25 / 1024 / 1024 / 1024 / 1024 as cost_usd
FROM `project.region-us.INFORMATION_SCHEMA.JOBS_BY_PROJECT`
WHERE creation_time >= TIMESTAMP_SUB(CURRENT_TIMESTAMP(), INTERVAL 30 DAY)
GROUP BY 1
ORDER BY 3 DESC
LIMIT 10
"""

# Top 10 usuarios por costo
for row in client.query(query):
    print(f"{row.user_email}: ${row.cost_usd:.2f}")
</code></pre>
<h2>Técnica #7: Cache de Resultados</h2>
<p>BigQuery cachea queries por 24hs. <strong>Aprovechalo.</strong></p>
<pre><code class="language-python"># Forzar uso de cache
job_config = bigquery.QueryJobConfig(use_query_cache=True)
results = client.query(query, job_config=job_config)
</code></pre>

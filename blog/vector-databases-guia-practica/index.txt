f:T1661,<h2>¿Por Qué Vector Databases?</h2>
<p>Los LLMs tienen un problema: <strong>su conocimiento está congelado en el tiempo de entrenamiento</strong>.</p>
<p>Para darles contexto actualizado (tus documentos, tu data), necesitás <strong>RAG: Retrieval Augmented Generation</strong>.</p>
<p>Y RAG necesita vector databases.</p>
<h2>¿Qué es un Vector Database?</h2>
<p>Almacena embeddings (vectores numéricos) de tus textos y permite búsqueda por similitud semántica.</p>
<pre><code class="language-python"># Texto → Embedding (vector)
text = "Cómo optimizar queries en BigQuery"
embedding = openai.Embedding.create(input=text, model="text-embedding-3-small")
# Output: [0.023, -0.154, 0.892, ..., 0.231]  # 1536 dimensiones
</code></pre>
<h2>El Pipeline RAG Completo</h2>
<pre><code class="language-python">import openai
import pinecone

# 1. Inicializar vector DB
pinecone.init(api_key="tu-api-key")
index = pinecone.Index("knowledge-base")

# 2. Indexar documentos
def index_documents(docs):
    for doc in docs:
        # Generate embedding
        embedding = openai.Embedding.create(
            input=doc["text"],
            model="text-embedding-3-small"
        )["data"][0]["embedding"]
        
        # Store in Pinecone
        index.upsert([(doc["id"], embedding, {"text": doc["text"]})])

# 3. Query con contexto
def query_with_context(question):
    # Generate embedding del question
    q_embedding = openai.Embedding.create(
        input=question,
        model="text-embedding-3-small"
    )["data"][0]["embedding"]
    
    # Buscar docs similares
    results = index.query(q_embedding, top_k=3, include_metadata=True)
    
    # Construir contexto
    context = "\n\n".join([r["metadata"]["text"] for r in results["matches"]])
    
    # LLM con contexto
    response = openai.ChatCompletion.create(
        model="gpt-4",
        messages=[
            {"role": "system", "content": f"Contexto:\n{context}"},
            {"role": "user", "content": question}
        ]
    )
    
    return response.choices[0].message.content
</code></pre>
<h2>Comparativa: Pinecone vs Weaviate vs Milvus</h2>
<h3>Pinecone (Cloud-Only)</h3>
<p><strong>Pros:</strong></p>
<ul>
<li>Setup en 5 minutos</li>
<li>Managed completamente</li>
<li>Performance excelente</li>
<li>SDKs en todo lenguaje</li>
</ul>
<p><strong>Contras:</strong></p>
<ul>
<li>Caro ($70/mes plan starter)</li>
<li>Vendor lock-in</li>
<li>No self-hosted</li>
</ul>
<p><strong>Ideal para:</strong> MVPs, startups, no quieres ops</p>
<h3>Weaviate (Cloud o Self-Hosted)</h3>
<p><strong>Pros:</strong></p>
<ul>
<li>Open-source</li>
<li>GraphQL API</li>
<li>Filtros complejos</li>
<li>Multi-tenancy</li>
</ul>
<p><strong>Contras:</strong></p>
<ul>
<li>Más complejo de configurar</li>
<li>Performance variable self-hosted</li>
</ul>
<p><strong>Ideal para:</strong> Empresas con DevOps team</p>
<h3>Milvus (Self-Hosted)</h3>
<p><strong>Pros:</strong></p>
<ul>
<li>100% open-source</li>
<li>Performance brutal (GPU support)</li>
<li>Escalabilidad masiva</li>
</ul>
<p><strong>Contras:</strong></p>
<ul>
<li>Requiere K8s en producción</li>
<li>Curva de aprendizaje alta</li>
</ul>
<p><strong>Ideal para:</strong> Big tech, millones de vectors</p>

<h2>Tips de Producción</h2>
<h3>1. Chunk Strategy</h3>
<pre><code class="language-python">def chunk_document(text, chunk_size=500, overlap=50):
    chunks = []
    for i in range(0, len(text), chunk_size - overlap):
        chunk = text[i:i + chunk_size]
        chunks.append(chunk)
    return chunks
</code></pre>
<p><strong>Por qué:</strong> Embeddings funcionan mejor en chunks de 200-800 tokens.</p>
<h3>2. Metadata Filtering</h3>
<pre><code class="language-python"># Buscar solo en documentos de 2026
results = index.query(

</code></pre>

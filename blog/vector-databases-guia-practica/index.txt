10:T278b,<p>Si estás implementando RAG (Retrieval Augmented Generation), tarde o temprano tenés que elegir una base de datos vectorial. Pinecone, Weaviate, Milvus, Qdrant, Chroma, pgvector... la cantidad de opciones es abrumadora.</p>
<p>Esta guía te da el conocimiento mínimo para entender qué son, cómo funcionan, y cuándo usar cada opción.</p>
<h2>¿Qué es una base de datos vectorial?</h2>
<p>Una base de datos vectorial almacena <strong>embeddings</strong> — representaciones numéricas de texto, imágenes, o cualquier dato — como vectores de alta dimensión (típicamente 768, 1536, o 3072 dimensiones).</p>
<p>La operación fundamental es la <strong>búsqueda de similitud</strong>: dado un vector de consulta, encontrá los N vectores más similares en la base de datos.</p>
<pre><code class="language-python"># Concepto simplificado
query = "¿Cuál es la política de devoluciones?"
query_embedding = embed(query)  # [0.23, -0.45, 0.12, ...]  ← 1536 números

# La base vectorial encuentra los documentos más "parecidos" semánticamente
documentos_relevantes = vector_db.search(
    query_vector=query_embedding,
    top_k=5
)
</code></pre>
<p>La magia es que "similitud" es <strong>semántica</strong>, no textual. "Cómo hago una devolución" y "quiero retornar un producto" encontrarán los mismos documentos aunque no compartan palabras.</p>
<h2>Cómo funciona el índice vectorial</h2>
<p>Las bases vectoriales usan algoritmos de Approximate Nearest Neighbor (ANN) para hacer búsquedas eficientes en millones de vectores:</p>
<ul>
<li><strong>HNSW</strong> (Hierarchical Navigable Small World): El más común, excelente balance velocidad/precisión</li>
<li><strong>IVF</strong> (Inverted File): Agrupa vectores en clusters, más eficiente para datasets gigantes</li>
<li><strong>LSH</strong> (Locality Sensitive Hashing): Más rápido pero menos preciso</li>
</ul>
<p>La diferencia clave con una búsqueda exacta: ANN sacrifica algo de precisión (puede no encontrar el MEJOR resultado, pero sí uno muy bueno) a cambio de velocidad.</p>
<h2>El pipeline RAG completo</h2>
<pre><code class="language-python">from openai import OpenAI
import pinecone
from typing import list

client = OpenAI()

def crear_embedding(texto: str) -> list[float]:
    """Convierte texto en vector usando OpenAI."""
    response = client.embeddings.create(
        model="text-embedding-3-small",
        input=texto
    )
    return response.data[0].embedding

def indexar_documentos(documentos: list[str], index):
    """Indexa documentos en Pinecone."""
    for i, doc in enumerate(documentos):
        embedding = crear_embedding(doc)
        index.upsert(vectors=[{
            "id": f"doc-{i}",
            "values": embedding,
            "metadata": {"texto": doc}
        }])

def buscar_contexto(query: str, index, top_k: int = 5) -> list[str]:
    """Busca documentos relevantes para la query."""
    query_embedding = crear_embedding(query)
    resultados = index.query(
        vector=query_embedding,
        top_k=top_k,
        include_metadata=True
    )
    return [r.metadata["texto"] for r in resultados.matches]

def responder_con_rag(query: str, index) -> str:
    """RAG completo: recuperar contexto + generar respuesta."""
    contextos = buscar_contexto(query, index)
    contexto_unido = "\n\n".join(contextos)
    
    respuesta = client.chat.completions.create(
        model="gpt-4o",
        messages=[
            {
                "role": "system",
                "content": f"""Respondé la pregunta basándote SOLO en el siguiente contexto.
Si la información no está en el contexto, decilo claramente.

CONTEXTO:
{contexto_unido}"""
            },
            {"role": "user", "content": query}
        ]
    )
    
    return respuesta.choices[0].message.content
</code></pre>
<h2>Comparativa: Pinecone vs Weaviate vs Qdrant vs pgvector</h2>
<h3>Pinecone</h3>
<p><strong>Pros:</strong></p>
<ul>
<li>Setup en 5 minutos, sin infraestructura que manejar</li>
<li>Performance excelente para millones de vectores</li>
<li>SDK limpio y bien documentado</li>
<li>Filtrado por metadata out-of-the-box</li>
</ul>
<p><strong>Contras:</strong></p>
<ul>
<li>Vendor lock-in total (SaaS only)</li>
<li>Costo alto a escala (puede superar $1000/mes fácilmente)</li>
<li>Sin control sobre la infraestructura</li>
</ul>
<pre><code class="language-python">from pinecone import Pinecone

pc = Pinecone(api_key="tu-api-key")
index = pc.Index("mi-indice")

# Insertar
index.upsert(vectors=[{
    "id": "vec1",
    "values": [0.1, 0.2, 0.3, ...],  # 1536 dimensiones
    "metadata": {"fuente": "doc1", "seccion": "intro"}
}])

# Buscar con filtro
resultados = index.query(
    vector=[0.1, 0.2, ...],
    top_k=10,
    filter={"fuente": {"$eq": "doc1"}},  # Solo documentos de "doc1"
    include_metadata=True
)
</code></pre>
<p><strong>Cuándo usar Pinecone</strong>: Prototipado rápido, startup sin equipo de infraestructura, cuando el costo no es un bloqueante.</p>
<h3>Weaviate</h3>
<p><strong>Pros:</strong></p>
<ul>
<li>Open source + managed cloud</li>
<li>Módulos built-in para vectorización (conecta directo con OpenAI, Cohere, etc.)</li>
<li>GraphQL API poderosa</li>
<li>Búsqueda híbrida (vectorial + keyword) out-of-the-box</li>
</ul>
<p><strong>Contras:</strong></p>
<ul>
<li>Curva de aprendizaje más alta</li>
<li>El schema es más complejo de definir</li>
</ul>
<pre><code class="language-python">import weaviate
from weaviate.classes.init import Auth

client = weaviate.connect_to_weaviate_cloud(
    cluster_url="tu-cluster.weaviate.network",
    auth_credentials=Auth.api_key("tu-api-key"),
    headers={"X-OpenAI-Api-Key": "tu-openai-key"},
)

# Weaviate puede vectorizar automáticamente
collection = client.collections.get("Documento")
collection.data.insert({
    "titulo": "Política de devoluciones",
    "contenido": "Los productos pueden devolverse en 30 días...",
})

# Búsqueda semántica
resultados = collection.query.near_text(
    query="cómo hago una devolución",
    limit=5
)
</code></pre>
<p><strong>Cuándo usar Weaviate</strong>: Cuando necesitás búsqueda híbrida, cuando querés vectorización automática integrada.</p>
<h3>Qdrant</h3>
<p><strong>Pros:</strong></p>
<ul>
<li>Open source, escrito en Rust (muy rápido)</li>
<li>Excelente para on-premise / self-hosted</li>
<li>Filtrado avanzado con payload</li>
<li>Docker compose en 2 minutos</li>
</ul>
<p><strong>Contras:</strong></p>
<ul>
<li>Menor ecosistema que Pinecone/Weaviate</li>
</ul>
<pre><code class="language-bash"># Levantar Qdrant localmente
docker run -p 6333:6333 qdrant/qdrant
</code></pre>
<pre><code class="language-python">from qdrant_client import QdrantClient
from qdrant_client.models import Distance, VectorParams, PointStruct

client = QdrantClient(host="localhost", port=6333)

# Crear colección
client.create_collection(
    collection_name="documentos",
    vectors_config=VectorParams(size=1536, distance=Distance.COSINE),
)

# Insertar
client.upsert(
    collection_name="documentos",
    points=[
        PointStruct(
            id=1,
            vector=[0.1, 0.2, ...],
            payload={"texto": "...", "fuente": "manual"}
        )
    ]
)

# Buscar
resultados = client.search(
    collection_name="documentos",
    query_vector=[0.1, 0.2, ...],
    limit=5
)
</code></pre>
<p><strong>Cuándo usar Qdrant</strong>: Self-hosted, performance crítica, cuando querés control total.</p>
<h3>pgvector (extensión de PostgreSQL)</h3>
<p><strong>Pros:</strong></p>
<ul>
<li>Ya tenés Postgres → zero infraestructura adicional</li>
<li>SQL que ya conocés</li>
<li>Transacciones ACID</li>
<li>Gratis si ya pagás por Postgres</li>
</ul>
<p><strong>Contras:</strong></p>
<ul>
<li>Performance menor que soluciones dedicadas a millones de vectores</li>
<li>Índices HNSW requieren ajuste fino</li>
</ul>
<pre><code class="language-sql">-- Habilitar extensión
CREATE EXTENSION IF NOT EXISTS vector;

-- Tabla con columna vectorial
CREATE TABLE documentos (
    id SERIAL PRIMARY KEY,
    texto TEXT,
    fuente VARCHAR(100),
    embedding vector(1536)
);

-- Índice HNSW para búsqueda eficiente
CREATE INDEX ON documentos USING hnsw (embedding vector_cosine_ops);

-- Insertar (desde Python)
-- INSERT INTO documentos (texto, fuente, embedding)
-- VALUES ('Política de devoluciones...', 'manual', '[0.1, 0.2, ...]')

-- Búsqueda de similitud coseno
SELECT texto, fuente, 1 - (embedding &#x3C;=> '[0.1, 0.2, ...]') as similitud
FROM documentos
ORDER BY embedding &#x3C;=> '[0.1, 0.2, ...]'
LIMIT 5;
</code></pre>
<p><strong>Cuándo usar pgvector</strong>: Dataset pequeño (&#x3C;1M vectores), ya usás PostgreSQL, simplicidad operacional es prioridad.</p>

<h2>Métricas de similitud: qué elegir</h2>
<p>Las bases vectoriales ofrecen distintas métricas de distancia:</p>
<ul>
<li><strong>Cosine similarity</strong>: Para texto y embeddings de lenguaje. Mide el ángulo entre vectores, no su magnitud. <strong>La más común para RAG.</strong></li>
<li><strong>Dot product</strong>: Similar a coseno pero magnitud importa. Bueno para embeddings normalizados.</li>
<li><strong>Euclidean (L2)</strong>: Para cuando la distancia absoluta importa (imágenes, coordenadas geográficas).</li>
</ul>
<p>Para RAG con OpenAI embeddings, siempre usá <strong>cosine similarity</strong>.</p>

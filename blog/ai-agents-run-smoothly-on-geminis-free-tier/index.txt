<h3>Introducción</h3>
<p>En el mundo actual, la automatización y la inteligencia artificial (AI) son herramientas cruciales para optimizar procesos y reducir costos. Como desarrollador solitario en Taiwán, he logrado automatizar completamente mi agencia tecnológica utilizando cuatro agentes AI que manejan contenido, ventas, seguridad y operaciones. Todo esto se ejecuta en el tier gratuito de Gemini 2.5 Flash, lo que me permite mantener un costo mensual de $0 en modelos de lenguaje (LLM).</p>
<h3>Arquitectura del Sistema</h3>
<h4>Componentes Principales</h4>
<ol>
<li><strong>Agentes AI</strong>: Cuatro agentes construidos con OpenClaw, una plataforma de agentes AI de código abierto.</li>
<li><strong>Entorno de Ejecución</strong>: Windows Subsystem for Linux 2 (WSL2) en mi máquina local, gestionado con systemd timers.</li>
<li><strong>Infraestructura</strong>: Vercel (hobby tier) y Firebase (free tier) para el hosting y la base de datos.</li>
<li><strong>APIs y Servicios</strong>: Telegram Bot, Resend, Jina Reader para tareas específicas como envío de mensajes y lectura de contenido web.</li>
</ol>
<h3>Funcionamiento Diario de los Agentes</h3>
<h4>Generación de Contenido Social</h4>
<ul>
<li>
<p><strong>Proceso</strong>:</p>
<ol>
<li><strong>Generación Inicial</strong>: Los agentes generan 8 publicaciones sociales diarias para diferentes plataformas.</li>
<li><strong>Auto-revisión</strong>: Cada publicación es revisada por el mismo agente para asegurar su calidad.</li>
<li><strong>Reescritura si Es Necesario</strong>: Si la puntuación de calidad es baja, el agente reescribe la publicación hasta alcanzar un estándar aceptable.</li>
</ol>
</li>
<li>
<p><strong>Código de Ejemplo</strong>:</p>
<pre><code class="language-python">def generate_social_post(platform):
    # Leer archivos de inteligencia locales
    intelligence = read_intelligence_files()
    
    # Generar publicación inicial
    initial_post = generate_post(intelligence)
    
    # Auto-revisión
    quality_score = review_post(initial_post)
    
    # Reescritura si es necesario
    while quality_score &#x3C; MIN_QUALITY_THRESHOLD:
        initial_post = rewrite_post(initial_post)
        quality_score = review_post(initial_post)
    
    return initial_post

def read_intelligence_files():
    # Leer archivos Markdown locales
    intelligence_files = glob.glob('intelligences/*.md')
    intelligence = {}
    for file in intelligence_files:
        with open(file, 'r') as f:
            intelligence[file] = f.read()
    return intelligence
</code></pre>

<h3>Optimización de Tokens</h3>
<p>Para maximizar la eficiencia y minimizar el uso de tokens, he implementado una estrategia de optimización:</p>
<ol>
<li><strong>Lectura de Archivos Locales</strong>: Los agentes leen archivos de inteligencia locales en formato Markdown, lo que no consume tokens.</li>
<li><strong>Prompts Focales</strong>: Cada solicitud de LLM incluye un prompt enfocado con todo el contexto necesario.</li>
<li><strong>Respuestas Concisas</strong>: Las respuestas de los agentes son procesadas y actuadas de manera inmediata, evitando conversaciones largas.</li>
</ol>
<ul>
<li><strong>Código de Ejemplo</strong>:
<pre><code class="language-python">def optimize_token_usage(prompt, context):

    # Construir prompt focal

    # Enviar solicitud a LLM
    response = send_to_llm(full_prompt)
    
    # Procesar y actuar
    action = parse_response(response)
    execute_action(action)

def send_to_llm(prompt):
    # Enviar solicitud a Gemini 2.5 Flash
    response = requests.post(GEMINI_API_URL, json={'prompt': prompt})

<h3>Problemas y Lecciones Aprendidas</h3>
<ul>
<li><strong>Facturación Inesperada</strong>: Creé una API key desde un proyecto GCP habilitado para facturación en lugar de AI Studio, lo que resultó en una factura de $127 en 7 días. La lección es siempre crear claves API directamente desde AI Studio.</li>
<li><strong>Bucle de Compromiso</strong>: Un error en el bucle de compromiso hizo que todos los posts fueran iterados en lugar de los top N, quemando 800 RPD en un día y dejando sin recursos a otras tareas.</li>
<li><strong>Conflicto de Actualizaciones de Telegram</strong>: Las verificaciones de salud de Telegram llamaban a <code>getUpdates</code>, lo que entraba en conflicto con el gateway de largo polling, resultando en 18 mensajes duplicados en 3 minutos.</li>

f:T1062,<h2>El Benchmark</h2>
<p>Dataset: 100 millones de filas de transacciones de MercadoLibre.</p>
<p>Operaciones comunes:</p>
<ul>
<li>Filtrado</li>

<p><strong>Hardware:</strong> MacBook Pro M2, 16GB RAM</p>
<h2>Test #1: Lectura de CSV</h2>
<h3>Pandas</h3>
<pre><code class="language-python">import pandas as pd
import time

start = time.time()
df = pd.read_csv('transactions_100m.csv')
print(f"Pandas: {time.time() - start:.2f}s")
# Output: Pandas: 87.3s
</code></pre>
<h3>Polars</h3>
<pre><code class="language-python">import polars as pl

start = time.time()
df = pl.read_csv('transactions_100m.csv')
print(f"Polars: {time.time() - start:.2f}s")
# Output: Polars: 12.4s
</code></pre>
<p><strong>Ganador: Polars (7x más rápido)</strong></p>
<h2>Test #2: Filtrado</h2>
<pre><code class="language-python"># Pandas
filtered = df[df['amount'] > 1000]
# Tiempo: 2.3s

# Polars
filtered = df.filter(pl.col('amount') > 1000)
# Tiempo: 0.18s
</code></pre>
<p><strong>Ganador: Polars (12x más rápido)</strong></p>
<h2>Test #3: Group By + Aggregation</h2>
<pre><code class="language-python"># Pandas
result = df.groupby('country_code').agg({
    'amount': ['sum', 'mean', 'count']
})
# Tiempo: 8.7s

# Polars
result = df.groupby('country_code').agg([
    pl.col('amount').sum(),
    pl.col('amount').mean(),
    pl.col('amount').count()
])
# Tiempo: 0.6s
</code></pre>
<p><strong>Ganador: Polars (14x más rápido)</strong></p>
<h2>Test #4: Join de Dos Tablas</h2>
<pre><code class="language-python"># Pandas (100M filas + 10M filas)
merged = pd.merge(df1, df2, on='user_id', how='left')

merged = df1.join(df2, on='user_id', how='left')
# Tiempo: 4.1s
# Memoria: 2.1GB
</code></pre>

<h2>Sintaxis Comparada</h2>
<h3>Pandas</h3>
<pre><code class="language-python">result = (df
    .query('amount > 1000')
    .groupby('country')['amount']
    .sum()
    .reset_index()
)
</code></pre>
<h3>Polars</h3>
<pre><code class="language-python">result = (df
    .filter(pl.col('amount') > 1000)
    .groupby('country')
    .agg(pl.col('amount').sum())
)
</code></pre>
<p>Polars es más explícito, pero <strong>más claro</strong>.</p>
<h2>¿Debería Migrar a Polars?</h2>
<h3>SÍ, migrá si:</h3>
<ul>
<li>✅ Trabajás con >10M filas regularmente</li>
<li>✅ Performance es crítico</li>
<li>✅ Tus pipelines son lentos</li>
<li>✅ Te quedás sin memoria</li>
</ul>
<h3>NO migres si:</h3>
<ul>
<li>❌ Dataset &#x3C; 1M filas (diferencia mínima)</li>
<li>❌ Tu código depende de features específicas de Pandas</li>
<li>❌ Team no tiene tiempo de aprender nueva API</li>
</ul>
<h2>Cómo Migrar Gradualmente</h2>
<ol>
<li><strong>Usa Polars para lectura</strong>: <code>pl.read_csv()</code> → <code>to_pandas()</code></li>
<li><strong>Procesa con Polars, exporta a Pandas</strong> si otros sistemas lo requieren</li>
<li><strong>Migrá pipeline por pipeline</strong>, no todo de golpe</li>
</ol>
<h2>Conclusión</h2>
<p>Polars es <strong>objetivamente superior en performance</strong>.</p>
<p>Para data engineering moderno, es el nuevo estándar.</p>
<p>Pandas sigue vivo para notebooks y análisis exploratorio, pero <strong>para producción, Polars gana</strong>.</p>

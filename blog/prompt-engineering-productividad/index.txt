12:T133f,<p>El prompt engineering no es una habilidad mística — es un conjunto de técnicas concretas que mejoran dramáticamente la calidad de las respuestas de los LLMs. Acá las que más impacto tienen en mi flujo de trabajo.</p>
<h2>Por qué importa el prompting</h2>
<p>Con el mismo modelo (GPT-4o, Claude 3.5), un prompt bien construido puede:</p>
<ul>
<li>Reducir alucinaciones en 60-70%</li>
<li>Mejorar la consistencia del formato de salida</li>
<li>Obtener respuestas más específicas y accionables</li>
<li>Reducir el número de iteraciones para llegar al resultado</li>
</ul>
<h2>Técnica 1: Asignación de rol con contexto específico</h2>
<pre><code>❌ Malo:
"Revisá este código Python."

✅ Bueno:
"Sos un Python senior engineer especializado en data pipelines y performance. 
Tenés 10 años de experiencia en sistemas de alta escala. Tu tarea es revisar 
el siguiente código e identificar:
1. Problemas de performance (memoria, CPU, I/O)
2. Posibles excepciones no manejadas
3. Antipatrones específicos de Python
4. Sugerencias con el fix concreto, no teoría general.

Código:
[código acá]"
</code></pre>
<h2>Técnica 2: Few-shot examples</h2>
<p>Los ejemplos son más poderosos que las instrucciones:</p>
<pre><code>Clasificá el siguiente ticket de soporte en una categoría.

Categorías disponibles: BILLING, TECHNICAL, ACCOUNT, DELIVERY, OTHER

Ejemplos:
Input: "No me llega el paquete, ya pasaron 5 días"
Output: DELIVERY

Input: "Me cobró dos veces el mismo pedido"
Output: BILLING

Input: "No puedo iniciar sesión con mi cuenta"
Output: ACCOUNT

Input: "La app se cierra cuando intento agregar al carrito"
Output: TECHNICAL

Ahora clasificá este:
Input: "Quiero cambiar mi contraseña porque la olvidé"
Output:
</code></pre>
<h2>Técnica 3: Chain of Thought (CoT)</h2>
<p>Para tareas de razonamiento, pedirle al modelo que piense paso a paso:</p>
<pre><code>Analizá si esta campaña de email marketing fue exitosa. 
Pensá paso a paso antes de dar tu conclusión.

Datos:
- Enviados: 50,000 emails
- Abiertos: 8,500 (17% open rate)
- Clicks: 1,200 (14.1% CTR)
- Conversiones: 48 (4% conversion rate)
- Revenue generado: $14,400
- Costo de la campaña: $2,800

Mostrá tu razonamiento antes de la conclusión final.
</code></pre>
<p>Esto produce análisis más profundos y con menos errores que pedir directamente "¿fue exitosa?".</p>
<h2>Técnica 4: Restricciones explícitas</h2>
<pre><code>Escribí un email para rechazar la propuesta de un proveedor.

RESTRICCIONES (obligatorio cumplir todas):
- Máximo 150 palabras
- Tono profesional pero cálido, no frío
- No mencionar el motivo específico del rechazo
- Dejar la puerta abierta para futuras colaboraciones
- En español neutro, no Argentina
- Primer párrafo: agradecimiento
- Segundo párrafo: decisión
- Tercer párrafo: futuro
</code></pre>
<h2>Técnica 5: Formato de salida explícito</h2>
<pre><code>Generá un análisis competitivo de las siguientes 3 empresas.

FORMATO DE SALIDA REQUERIDO:
## [Nombre Empresa]
**Fortalezas**: [lista con bullets]
**Debilidades**: [lista con bullets]
**Oportunidad para nosotros**: [1-2 oraciones]
**Score competitivo**: [número del 1 al 10]

No uses otro formato. No agregues introducción ni conclusión.
</code></pre>
<h2>Técnica 6: Iteración con feedback</h2>
<p>En lugar de escribir el prompt perfecto de entrada, usar un ciclo de mejora:</p>
<pre><code>Paso 1: "Resumí este documento en 3 bullets"
Paso 2: "El tercer bullet está incompleto. Expandilo con más detalle sobre X"
Paso 3: "Ahora reescribí los 3 bullets en formato ejecutivo, más concreto"
</code></pre>
<h2>Técnica 7: Delimiter para separar instrucciones de datos</h2>
<pre><code>Clasificá el texto delimitado por triple comillas según el sentimiento.

Respuesta: solo "POSITIVO", "NEGATIVO" o "NEUTRO". Sin explicación.

"""
El producto llegó antes de lo esperado y la calidad superó mis expectativas. 
Definitivamente volvería a comprar.
"""
</code></pre>
<p>Los delimitadores (```, ----, ===) previenen que el modelo confunda las instrucciones con el contenido.</p>
<h2>Template para tareas de código</h2>
<pre><code>CONTEXTO: [Describe el sistema/proyecto en 2-3 líneas]

TAREA: [Qué exactamente necesitás que haga]

CONSTRAINS:
- [Restricción técnica 1]
- [Restricción técnica 2]

CÓDIGO EXISTENTE:
```python
[código acá]
</code></pre>
<p>SALIDA ESPERADA: [Describe el output exacto que esperás]</p>
<p>NOTA IMPORTANTE: [Cualquier context que el modelo no debe ignorar]</p>
<pre><code>
## Herramientas para iterar prompts

- **PromptFlow** (Azure): Testing sistemático de prompts
- **LangSmith**: Trazabilidad de chains de LangChain
- **Promptfoo**: Open source, CI/CD para prompts
- **Weights &#x26; Biases Prompts**: Para ML workflows

El prompt engineering es una habilidad que mejora con práctica. La clave: iterar rápido, medir el output, y documentar qué funciona.
</code></pre>

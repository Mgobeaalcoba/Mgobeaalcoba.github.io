Antes de usarlo, hay conceptos fundamentales que hay que entender bien. Acá la guía práctica para data engineers.</p>
<h2>¿Qué problema resuelve Kafka?</h2>
<p>Sin Kafka, los sistemas se comunican directamente:</p>
<pre><code>Service A → API Call → Service B → API Call → Service C
</code></pre>
<p>El problema: si B está caído, A no puede procesar. Si C es lento, bloquea todo. No hay buffer, no hay replay.</p>
<p>Con Kafka:</p>
<pre><code>Service A → Kafka Topic → Service B
                       → Service C  
                       → Service D (replay histórico)
</code></pre>
<p>Kafka desacopla productores de consumidores y persiste mensajes para replay.</p>
<h2>Conceptos clave</h2>
<p><strong>Topic</strong>: Un canal de mensajes. Como una tabla en una base de datos, pero append-only.</p>
<p><strong>Partition</strong>: Un topic se divide en particiones para paralelismo. Cada partición es una secuencia ordenada de mensajes.</p>
<p><strong>Offset</strong>: La posición de un mensaje dentro de una partición. Los consumers trackean su offset para saber qué procesaron.</p>
<p><strong>Consumer Group</strong>: Un grupo de consumers que dividen las particiones de un topic. Permite escalar horizontalmente el consumo.</p>
<h2>Productor en Python</h2>
<pre><code class="language-python">from kafka import KafkaProducer
import json
from datetime import datetime

producer = KafkaProducer(
    bootstrap_servers=["kafka-broker:9092"],
    value_serializer=lambda v: json.dumps(v).encode("utf-8"),
    key_serializer=lambda k: k.encode("utf-8") if k else None,
    acks="all",        # Esperar confirmación de todos los replicas
    retries=3,
    max_in_flight_requests_per_connection=1  # Orden garantizado
)

def publish_order_event(order: dict):
    event = {
        **order,
        "event_type": "order_created",
        "timestamp": datetime.utcnow().isoformat(),
    }
    
    future = producer.send(
        topic="orders",
        key=str(order["seller_id"]),  # Misma key → misma partición → orden garantizado
        value=event,
    )
    
    # Confirmar que el mensaje fue enviado
    record_metadata = future.get(timeout=10)
    return record_metadata.offset

producer.flush()
producer.close()
</code></pre>
<h2>Consumer en Python</h2>
<pre><code class="language-python">from kafka import KafkaConsumer
import json

consumer = KafkaConsumer(
    "orders",

    value_deserializer=lambda v: json.loads(v.decode("utf-8")),
    auto_offset_reset="earliest",    # Si es nuevo consumer group, empieza desde el principio
    enable_auto_commit=False,        # Commit manual para control exacto
    max_poll_records=500,
)

def process_batch(messages: list) -> bool:
    """Procesar un batch de mensajes. Devuelve True si todo salió bien."""
    try:
        for msg in messages:
            order = msg.value
            # Tu lógica de negocio acá
            enrich_and_load(order)
        return True
    except Exception as e:
        logger.error(f"Batch processing failed: {e}")
        return False

for message in consumer:
    success = process_batch([message])
    
    if success:
        # Solo commitear si el procesamiento fue exitoso
        consumer.commit()
    else:
        # No commitear: el mensaje se va a re-procesar
        logger.error("Message not committed, will retry")

<h2>Dead Letter Queue</h2>
<pre><code class="language-python">MAIN_TOPIC = "orders"
DLQ_TOPIC = "orders-dlq"

def consume_with_dlq():
    for message in consumer:
        try:
            process(message.value)
            consumer.commit()
        except NonRetryableError as e:
            # Error irrecuperable: mover al DLQ
            producer.send(DLQ_TOPIC, value={
                "original_message": message.value,
                "error": str(e),
                "failed_at": datetime.utcnow().isoformat(),
                "topic": message.topic,
                "partition": message.partition,
                "offset": message.offset,
            })
            consumer.commit()  # Commitear para no repetir
        except RetryableError:
            # No commitear: Kafka va a re-entregar
            pass

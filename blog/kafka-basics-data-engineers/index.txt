f:T1609,<h2>¿Qué Es Kafka en Una Frase?</h2>
<p><strong>Sistema de mensajería distribuido y ultra-rápido</strong> para mover datos en tiempo real entre sistemas.</p>
<h2>Conceptos Clave</h2>
<h3>1. Topics</h3>
<p>Categorías donde se publican mensajes.</p>
<pre><code class="language-python"># Publicar mensaje a topic "transactions"
producer.send('transactions', value={'user_id': 123, 'amount': 5000})
</code></pre>
<h3>2. Particiones</h3>
<p>Topics divididos en particiones para <strong>paralelismo</strong>.</p>
<pre><code>Topic: transactions (3 partitions)
├── Partition 0: [msg1, msg4, msg7, ...]
├── Partition 1: [msg2, msg5, msg8, ...]
└── Partition 2: [msg3, msg6, msg9, ...]
</code></pre>
<h3>3. Consumer Groups</h3>
<p>Múltiples consumers procesando el mismo topic en paralelo.</p>
<pre><code class="language-python"># Consumer 1 lee partitions 0-1
# Consumer 2 lee partition 2
# = 2x throughput
</code></pre>
<h2>Producer Example (Python)</h2>
<pre><code class="language-python">from kafka import KafkaProducer
import json

producer = KafkaProducer(
    bootstrap_servers=['localhost:9092'],
    value_serializer=lambda v: json.dumps(v).encode('utf-8')
)

# Enviar mensaje
producer.send('transactions', {
    'transaction_id': '123abc',
    'user_id': 456,
    'amount': 5000,
    'timestamp': '2026-05-03T10:30:00Z'
})

producer.flush()
</code></pre>
<h2>Consumer Example</h2>
<pre><code class="language-python">from kafka import KafkaConsumer

consumer = KafkaConsumer(
    'transactions',
    bootstrap_servers=['localhost:9092'],
    group_id='analytics-group',
    value_deserializer=lambda m: json.loads(m.decode('utf-8'))
)

for message in consumer:
    data = message.value
    print(f"Processing transaction {data['transaction_id']}")
    # Tu lógica de procesamiento aquí
</code></pre>
<h2>Kafka en Arquitectura Real</h2>
<pre><code>[Web App] ─┬─> Kafka Topic: "events"
[Mobile]  ─┤
[API]     ─┘
           │
           ├─> Consumer Group 1 (Real-time analytics)
           ├─> Consumer Group 2 (BigQuery sink)
           ├─> Consumer Group 3 (ML predictions)
           └─> Consumer Group 4 (Alerting system)
</code></pre>
<p><strong>Un mensaje → Múltiples consumers independientes.</strong></p>
<h2>Cuándo Usar Kafka</h2>
<h3>✅ Casos de Uso Ideales</h3>
<ol>
<li><strong>Event Streaming</strong>: Trackear eventos de usuario</li>
<li><strong>Log Aggregation</strong>: Centralizar logs de múltiples servicios</li>
<li><strong>CDC (Change Data Capture)</strong>: Replicar cambios de DB</li>
<li><strong>Microservices Communication</strong>: Desacoplar servicios</li>
</ol>
<h3>❌ Cuándo NO Usar Kafka</h3>
<ol>
<li><strong>Baja volumetría</strong>: &#x3C;100 mensajes/seg (overkill)</li>
<li><strong>No necesitás replay</strong>: Queue simple alcanza</li>
<li><strong>Team chico sin DevOps</strong>: Complejidad operacional alta</li>
<li><strong>Latencia ultra-baja</strong>: Redis Streams puede ser mejor</li>
</ol>

<h2>Errores Comunes</h2>
<h3>1. No Configurar Partitions Key</h3>
<pre><code class="language-python"># ❌ Sin key: mensajes en particiones random
producer.send('topic', value=data)

# ✅ Con key: mismo user siempre en misma partition (orden garantizado)
producer.send('topic', key=str(user_id).encode(), value=data)
</code></pre>
<h3>2. No Hacer Commit</h3>
<pre><code class="language-python"># ❌ Auto-commit puede perder mensajes
consumer = KafkaConsumer(enable_auto_commit=True)

# ✅ Commit manual después de procesar
consumer = KafkaConsumer(enable_auto_commit=False)

for message in consumer:
    process(message.value)
    consumer.commit()  # Solo commitear si procesamiento exitoso
</code></pre>
<h3>3. Ignorar Backpressure</h3>
<pre><code class="language-python"># ✅ Limitar batch size
consumer = KafkaConsumer(
    max_poll_records=100,  # Procesar 100 mensajes máximo por poll
    max_poll_interval_ms=300000  # 5 minutos para procesar batch
)
</code></pre>
<h2>Setup Rápido con Docker</h2>
<pre><code class="language-yaml">version: '3'
services:
  zookeeper:
    image: confluentinc/cp-zookeeper:7.5.0
    environment:
      ZOOKEEPER_CLIENT_PORT: 2181
  
  kafka:
    image: confluentinc/cp-kafka:7.5.0
    depends_on:
      - zookeeper
    ports:
      - "9092:9092"
    environment:
      KAFKA_BROKER_ID: 1
      KAFKA_ZOOKEEPER_CONNECT: zookeeper:2181
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://localhost:9092
</code></pre>

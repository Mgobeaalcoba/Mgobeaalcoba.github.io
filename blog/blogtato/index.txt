12:T1ba3,<h2>Introducción</h2>
<p>En el mundo del desarrollo de software, la gestión eficiente de información es crucial. Las fuentes de noticias como RSS y Atom feeds son herramientas valiosas para mantenerse actualizado con las últimas noticias y actualizaciones en diversos campos. Sin embargo, la gestión manual de estos feeds puede ser tediosa y poco eficiente. En este artículo, exploraremos un proyecto llamado <strong>blogtato</strong>, una CLI (Interfaz de Línea de Comandos) para leer feeds RSS/Atom inspirada en Taskwarrior y sincronizada usando Git.</p>
<h3>¿Qué es blogtato?</h3>
<p><strong>blogtato</strong> es una herramienta de línea de comandos diseñada para leer feeds RSS y Atom. Su inspiración proviene de <strong>Taskwarrior</strong>, una herramienta de gestión de tareas altamente personalizable y potente. La principal ventaja de <strong>blogtato</strong> es que permite la sincronización de feeds entre múltiples dispositivos utilizando Git, lo que facilita la gestión de información en entornos multiplataforma.</p>
<h2>Instalación y Configuración</h2>
<h3>Requisitos</h3>
<p>Antes de instalar <strong>blogtato</strong>, asegúrate de tener los siguientes requisitos cumplidos:</p>
<ul>
<li><strong>Python 3.8+</strong>: <strong>blogtato</strong> está escrito en Python y requiere una versión reciente.</li>
<li><strong>Git</strong>: Para la sincronización de feeds.</li>
<li><strong>feedparser</strong>: Una biblioteca Python para analizar feeds RSS y Atom.</li>
<li><strong>requests</strong>: Una biblioteca Python para realizar solicitudes HTTP.</li>
</ul>
<h3>Instalación</h3>
<p>Puedes instalar <strong>blogtato</strong> utilizando pip:</p>
<pre><code class="language-bash">pip install blogtato
</code></pre>
<p>Si prefieres instalar desde el repositorio de GitHub, puedes hacerlo de la siguiente manera:</p>
<pre><code class="language-bash">git clone https://github.com/kantord/blogtato.git
cd blogtato
pip install .
</code></pre>
<h3>Configuración Inicial</h3>
<p>Después de instalar <strong>blogtato</strong>, debes configurarlo para que pueda acceder a tus feeds y sincronizarlos con Git.</p>
<ol>
<li>
<p><strong>Crear el archivo de configuración</strong>:</p>
<pre><code class="language-bash">blogtato init
</code></pre>
<p>Esto creará un directorio <code>.blogtato</code> en tu directorio home con un archivo <code>config.yaml</code> y un directorio <code>feeds</code>.</p>
</li>
<li>
<p><strong>Configurar feeds</strong>:
Edita el archivo <code>config.yaml</code> para agregar tus feeds. Por ejemplo:</p>
<pre><code class="language-yaml">feeds:
  - url: https://example.com/rss.xml
    name: Example Feed
  - url: https://another-example.com/atom.xml
    name: Another Example Feed
</code></pre>
</li>
<li>
<p><strong>Inicializar el repositorio Git</strong>:</p>
<pre><code class="language-bash">cd ~/.blogtato
git init
git add .
git commit -m "Initial commit"
git remote add origin &#x3C;your-git-repo-url>
git push -u origin main
</code></pre>
</li>
</ol>
<h2>Uso Básico</h2>
<h3>Leer Feeds</h3>
<p>Para leer los feeds configurados, simplemente ejecuta:</p>
<pre><code class="language-bash">blogtato read
</code></pre>
<p>Esto mostrará una lista de los artículos más recientes de todos los feeds configurados. Puedes navegar por los artículos y marcarlos como leídos.</p>
<h3>Marcar Artículos como Leídos</h3>
<p>Para marcar un artículo como leído, usa el comando <code>mark</code> seguido del ID del artículo:</p>
<pre><code class="language-bash">blogtato mark &#x3C;article_id>
</code></pre>
<h3>Sincronización con Git</h3>
<p>La sincronización con Git es automática cada vez que realizas cambios en los feeds o marcas artículos como leídos. Sin embargo, puedes forzar la sincronización manualmente con:</p>
<pre><code class="language-bash">blogtato sync
</code></pre>
<h2>Características Avanzadas</h2>
<h3>Personalización de la Interfaz</h3>
<p><strong>blogtato</strong> permite personalizar la interfaz de línea de comandos. Puedes modificar el archivo <code>config.yaml</code> para cambiar el formato de salida, el número de artículos mostrados, y otros ajustes.</p>
<p>Por ejemplo, para cambiar el formato de salida de los artículos:</p>
<pre><code class="language-yaml">output_format: "{title} - {author} ({published})"
</code></pre>
<h3>Integración con Otros Servicios</h3>
<p><strong>blogtato</strong> puede integrarse con otros servicios mediante la extensión de su funcionalidad. Por ejemplo, puedes escribir scripts personalizados para enviar notificaciones a Slack cuando se publique un nuevo artículo en un feed específico.</p>
<h3>Ejemplo de Script de Notificación</h3>
<pre><code class="language-python">import requests
from blogtato import FeedManager

def send_slack_notification(article):
    url = "https://slack.com/api/chat.postMessage"
    headers = {
        "Authorization": f"Bearer {SLACK_BOT_TOKEN}",
        "Content-Type": "application/json"
    }
    data = {
        "channel": "#news",
        "text": f"New article: {article.title} - {article.author} ({article.published})"
    }
    response = requests.post(url, headers=headers, json=data)
    if response.status_code != 200:
        raise ValueError(f"Request to Slack returned an error {response.status_code}, the response is:\n{response.text}")

if __name__ == "__main__":
    manager = FeedManager()
    new_articles = manager.get_new_articles()
    for article in new_articles:
        send_slack_notification(article)
</code></pre>

<p><strong>blogtato</strong> es una herramienta poderosa y flexible para la gestión de feeds RSS y Atom. Su integración con Git facilita la sincronización entre múltiples dispositivos, y su diseño inspirado en Taskwarrior ofrece una experiencia de usuario familiar y eficiente. Ya sea para mantenerse al día con las últimas noticias o para gestionar feeds específicos de tu industria, <strong>blogtato</strong> es una opción sólida y altamente recomendable.</p>

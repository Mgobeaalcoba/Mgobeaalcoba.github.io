d:T14da,<h2>Por Qué Docker es Mandatorio en 2026</h2>
<p><strong>"Works on my machine"</strong> dejó de ser excusa hace años.</p>
<p>Si tu pipeline corre en tu laptop pero no en producción, <strong>el problema sos vos, no el código</strong>.</p>
<h2>Dockerfile para Data Pipeline</h2>
<pre><code class="language-dockerfile">FROM python:3.11-slim

# Instalar dependencias del sistema
RUN apt-get update &#x26;&#x26; apt-get install -y \
    gcc \
    g++ \
    &#x26;&#x26; rm -rf /var/lib/apt/lists/*

# Working directory
WORKDIR /app

# Copy requirements
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy código
COPY . .

# Run pipeline
CMD ["python", "pipeline.py"]
</code></pre>
<h2>docker-compose para Local Development</h2>
<pre><code class="language-yaml">version: '3.8'

services:
  postgres:
    image: postgres:15
    environment:
      POSTGRES_DB: analytics
      POSTGRES_USER: user
      POSTGRES_PASSWORD: pass
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

  pipeline:
    build: .
    depends_on:
      - postgres
    environment:
      DB_HOST: postgres
      DB_PORT: 5432
    volumes:
      - ./data:/app/data

volumes:
  postgres_data:
</code></pre>
<p>Levantar todo:</p>
<pre><code class="language-bash">docker-compose up -d
</code></pre>
<h2>Comandos Esenciales</h2>
<pre><code class="language-bash"># Construir imagen
docker build -t mi-pipeline:latest .

# Correr container
docker run -d --name pipeline mi-pipeline:latest

# Ver logs
docker logs -f pipeline

# Entrar al container (debugging)
docker exec -it pipeline bash

# Ver containers corriendo
docker ps

# Parar y eliminar
docker stop pipeline
docker rm pipeline

# Limpiar imágenes viejas
docker image prune -a

# Ver uso de recursos
docker stats
</code></pre>
<h2>Multi-Stage Build (Optimización)</h2>
<pre><code class="language-dockerfile"># Stage 1: Build
FROM python:3.11 as builder

WORKDIR /app
COPY requirements.txt .
RUN pip install --user --no-cache-dir -r requirements.txt

# Stage 2: Runtime (más chica)
FROM python:3.11-slim

WORKDIR /app

# Copy solo lo necesario del stage anterior
COPY --from=builder /root/.local /root/.local
COPY . .

# Asegurar que scripts estén en PATH
ENV PATH=/root/.local/bin:$PATH

CMD ["python", "pipeline.py"]
</code></pre>
<p><strong>Beneficio:</strong> Imagen final 60% más chica.</p>
<h2>.dockerignore (Esencial)</h2>
<pre><code class="language-.dockerignore"># Python
__pycache__/
*.pyc
*.pyo
*.pyd
.Python
venv/
.venv/

# Data (no querés esto en imagen)
*.csv
*.parquet
*.json
data/

# Git
.git/
.gitignore

# IDE
.vscode/
.idea/
*.swp

# Secrets
.env
*.key
credentials.json
</code></pre>
<h2>Volúmenes para Data</h2>
<pre><code class="language-bash"># ❌ Mal: Data en imagen (pesado, no persiste)
COPY data/ /app/data/

# ✅ Bien: Data en volumen
docker run -v $(pwd)/data:/app/data mi-pipeline
</code></pre>
<h2>Secrets Management</h2>
<pre><code class="language-python"># ❌ MAL: Hardcoded
API_KEY = "sk-abc123..."

# ❌ MAL: En imagen
ENV API_KEY=sk-abc123

# ✅ BIEN: Environment variables en runtime
docker run -e API_KEY=$API_KEY mi-pipeline

# ✅ MEJOR: Secrets manager
docker run --env-file .env mi-pipeline
</code></pre>
<h2>Health Checks</h2>
<pre><code class="language-dockerfile">HEALTHCHECK --interval=30s --timeout=3s \
  CMD python -c "import requests; requests.get('http://localhost:8000/health')" || exit 1
</code></pre>
<h2>Caso Real: Pipeline en MercadoLibre</h2>
<p>Antes de Docker:</p>
<ul>
<li>Setup local: 2 horas</li>
<li>"Works on my machine" bugs: Frecuentes</li>
<li>Deploy a producción: Manual, frágil</li>
</ul>
<p>Después de Docker:</p>
<ul>
<li>Setup local: 5 minutos (<code>docker-compose up</code>)</li>
<li>Consistency: 100%</li>
<li>Deploy: <code>docker push</code> + K8s auto-deploy</li>
</ul>
<h2>Docker vs Alternativas</h2>
<p>| Tool | Use Case | Learning Curve |
|------|----------|----------------|
| <strong>Docker</strong> | General purpose | ⭐⭐⭐ |
| <strong>Conda</strong> | Python-specific | ⭐⭐⭐⭐ |
| <strong>Virtualenv</strong> | Python dev only | ⭐⭐⭐⭐⭐ |
| <strong>VM</strong> | Full isolation | ⭐⭐ |</p>
<h2>Tips Pro</h2>
<h3>1. Layer Caching</h3>
<pre><code class="language-dockerfile"># ❌ Malo: requirements se invalida con cada código change
COPY . .
RUN pip install -r requirements.txt

# ✅ Bueno: requirements cachea unless cambie
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
</code></pre>
<h3>2. Non-Root User</h3>
<pre><code class="language-dockerfile"># Security best practice
RUN adduser --disabled-password --gecos '' appuser
USER appuser
</code></pre>
<h3>3. Logging a STDOUT</h3>
<pre><code class="language-python"># Para que docker logs funcione
import logging

logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
    handlers=[logging.StreamHandler()]  # STDOUT
)
</code></pre>

d:T1592,<h2>Introducción</h2>
<p>Después de 6+ años construyendo pipelines de datos en MercadoLibre, aprendí que <strong>los errores más costosos son los que parecen inofensivos al principio</strong>.</p>
<p>Estos son los 5 errores que veo repetirse constantemente y cómo evitarlos.</p>
<h2>Error #1: No Manejar Idempotencia</h2>
<h3>El Problema</h3>
<pre><code class="language-python"># ❌ Pipeline NO idempotente
def process_daily_sales():
    sales = fetch_sales_from_api()
    insert_into_database(sales)  # Duplica datos si se re-ejecuta
</code></pre>
<p>Si el pipeline falla y lo re-ejecutás, <strong>duplicás todos los datos</strong>.</p>
<h3>La Solución</h3>
<pre><code class="language-python"># ✅ Pipeline idempotente
def process_daily_sales(date):
    sales = fetch_sales_from_api(date)
    
    # DELETE antes de INSERT (o usa MERGE)
    delete_existing_data(date)
    insert_into_database(sales)
</code></pre>
<p><strong>Regla de oro:</strong> Tu pipeline debe poder ejecutarse N veces y el resultado debe ser el mismo.</p>
<h2>Error #2: Hardcodear Fechas</h2>
<h3>El Problema</h3>
<pre><code class="language-python"># ❌ Fechas hardcodeadas
sales = fetch_sales(start_date='2026-01-01', end_date='2026-01-31')
</code></pre>
<p>El código funciona en enero, pero <strong>explota en febrero</strong>.</p>
<h3>La Solución</h3>
<pre><code class="language-python"># ✅ Fechas dinámicas
from datetime import datetime, timedelta

today = datetime.now()
yesterday = today - timedelta(days=1)

sales = fetch_sales(
    start_date=yesterday.strftime('%Y-%m-%d'),
    end_date=today.strftime('%Y-%m-%d')
)
</code></pre>
<p><strong>Mejor aún:</strong> Usa variables de entorno o parámetros del scheduler.</p>
<h2>Error #3: No Validar Datos de Entrada</h2>
<h3>El Problema</h3>
<p>Asumís que los datos siempre vienen bien:</p>
<pre><code class="language-python"># ❌ Sin validación
for record in api_response:
    amount = float(record['amount'])  # BOOM si amount es None
</code></pre>
<h3>La Solución</h3>
<pre><code class="language-python"># ✅ Con validación robusta
import pandas as pd
from pydantic import BaseModel, validator

class SaleRecord(BaseModel):
    transaction_id: str
    amount: float
    currency: str = 'ARS'
    
    @validator('amount')
    def amount_must_be_positive(cls, v):
        if v &#x3C;= 0:
            raise ValueError('Amount must be positive')
        return v

# Usar Pydantic o Pandas para validar
validated_data = [SaleRecord(**record) for record in api_response]
</code></pre>
<p><strong>Resultado:</strong> Los errores se detectan temprano, no en producción.</p>
<h2>Error #4: No Monitorear el Pipeline</h2>
<h3>El Problema</h3>
<p>El pipeline corre silenciosamente... hasta que explota y <strong>nadie se entera por horas</strong>.</p>
<h3>La Solución</h3>
<pre><code class="language-python"># ✅ Logging y alertas
import logging
from datetime import datetime

logger = logging.getLogger(__name__)

def run_pipeline():
    start_time = datetime.now()
    records_processed = 0
    
    try:
        logger.info(f"Pipeline started at {start_time}")
        
        data = fetch_data()
        records_processed = len(data)
        process_data(data)
        
        duration = (datetime.now() - start_time).total_seconds()
        logger.info(f"Pipeline completed: {records_processed} records in {duration}s")
        
        # Enviar métricas a monitoring
        send_metrics({
            'records': records_processed,
            'duration_seconds': duration,
            'status': 'success'
        })
        
    except Exception as e:
        logger.error(f"Pipeline failed: {e}")
        send_alert(f"Pipeline failure: {e}")
        raise
</code></pre>
<p><strong>Herramientas recomendadas:</strong> Datadog, Grafana, CloudWatch</p>
<h2>Error #5: No Testear con Datos Reales</h2>
<h3>El Problema</h3>
<p>Tu pipeline funciona perfecto con 100 registros de prueba, pero <strong>colapsa con 10 millones en producción</strong>.</p>
<h3>La Solución</h3>
<pre><code class="language-python"># ✅ Testing con volúmenes reales
import pytest

@pytest.mark.parametrize("size", [100, 10_000, 1_000_000])
def test_pipeline_scalability(size):
    """Test pipeline con diferentes volúmenes"""
    data = generate_mock_data(size)
    
    start = time.time()
    result = process_pipeline(data)
    duration = time.time() - start
    
    # Validar performance
    assert duration &#x3C; size * 0.001  # &#x3C;1ms por registro
    assert len(result) == size
    assert result['errors'] == 0
</code></pre>
<p><strong>Pro tip:</strong> Crea datasets sintéticos que simulen casos extremos (nulls, duplicados, caracteres especiales).</p>
<h2>Bonus: Checklist de Pipeline Production-Ready</h2>
<p>Antes de deployar a producción:</p>
<ul>
<li>✅ Es idempotente?</li>
<li>✅ Tiene logging completo?</li>
<li>✅ Valida datos de entrada?</li>
<li>✅ Maneja excepciones correctamente?</li>
<li>✅ Tiene alertas configuradas?</li>
<li>✅ Se testeó con volúmenes reales?</li>
<li>✅ Tiene rollback strategy?</li>
<li>✅ Documenta sus dependencias?</li>
</ul>
